# Analysis Framework: Skill Range Check

Use this to evaluate a game's accessibility and depth.

## Step 1: Identify the Skill Barrier
What must a player already know/be able to do to start playing?
- Physical skills (mouse/keyboard, controller, touchscreen)
- Genre knowledge (FPS conventions, RTS conventions)
- Domain knowledge (reading, math, cultural context)
- **Rate**: How many potential players are excluded by this barrier?

## Step 2: Identify the Skill Ceiling
What does theoretically perfect play look like? Can any human achieve it?
- If perfect play is humanly achievable → finite depth (will eventually be mastered)
- If perfect play exceeds human ability → **limitlessly deep** (always more to learn)
- What creates the ceiling? (Reaction time limits, combinatorial complexity, opponent unpredictability)

## Step 3: Map the Reinventions
How does the game transform as skill increases?

| Skill Level | Reinvention | What the Game Is About |
|-------------|-------------|----------------------|
| Beginner | Manual | Mastering the interface (aiming, moving, basic controls) |
| Intermediate | Situational | Knowing what to do and when (map control, counterstrategies) |
| Advanced | Mental | Prediction, concentration, psychological manipulation |

- Does the game actually reinvent itself, or does it stay the same at all skill levels?
- How many internal reinventions exist within the situational level?

## Step 4: Check for Elastic Challenges
- Are challenges pass/fail or do they permit degrees of success?
- Can a beginner get *some* reward while an expert chases perfection?
- Examples: scoring systems, rating systems, optional objectives, speedrun potential

## Step 5: Evaluate Failure Handling
- [ ] Does the player understand why they failed?
- [ ] Can they try again quickly?
- [ ] Is failure a learning opportunity or just punishment?
- [ ] Are there **failure traps** (failure making the next attempt harder)?
- [ ] Is there **emotional life support** during failure (humor, spectacle, encouragement)?

## Step 6: Difficulty Modification
How does the game accommodate different skill levels?
- Self-selected difficulty levels
- Dynamic difficulty adjustment
- Elastic challenges built into the design
- Matchmaking (multiplayer)
- Assist modes / accessibility options

## Output Format
- **Skill barrier**: [description and severity]
- **Skill ceiling**: [finite/limitless, what creates it]
- **Skill range width**: [narrow/medium/wide]
- **Reinventions**: [list of how the game transforms]
- **Elastic challenges**: [present/absent, examples]
- **Failure handling**: [quality assessment]
- **Recommendations**: How to widen the skill range
