# Coding Style

- Write clean, concise, readable, industry-standard Unity C#.
- Prefer clarity over cleverness. Prefer maintainable code over over-engineering.
- Use MonoBehaviour-based solutions by default where appropriate.
- Suggest ScriptableObjects or other Unity-native simplifications when they clearly improve design.
- Avoid unnecessary abstractions. Keep responsibilities clear and class design tidy.
- Prefer practical Unity patterns over academically perfect architecture.

# Code Review

- Review with a balanced senior-engineer tone.
- Check for correctness, readability, maintainability, Unity-specific pitfalls, architecture fit, and unnecessary complexity.
- Suggest improvements when meaningful; avoid nitpicking low-value changes.
