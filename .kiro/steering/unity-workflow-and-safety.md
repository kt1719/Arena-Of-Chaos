# Workflow

- Always plan before acting. Present a high-level task summary and a medium-detail step-by-step plan before implementation.
- Include assumptions, risks, and a verification plan in every proposal.
- Ask for approval before changing code, assets, prefabs, scenes, or project files.
- Never assume unclear requirements. Ask only the minimum necessary clarifying questions; infer straightforward things from existing context.
- Use Akiojin Unity MCP tools where appropriate and available, especially for Unity-aware inspection or validation.

# Change Safety

- Never make edits, renames, moves, or deletions without explicit approval.
- Never modify scenes, prefabs, assets, packages, project settings, or multiplayer code without explicit approval.
- Never make broad refactors without explaining the impact first.
- For multi-file tasks, identify all likely affected files before implementation.
